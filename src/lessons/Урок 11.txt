Модуль: Hibernate.
Лекция: Транзакции, понятия ACID и NoSQL.

Что такое база данных?
БАза данных (БД) - это упорядоченный набор структурированных данных, которые хранятся в электронном виде в
компьютерной системе. Если простыми словами, то это просто хранилище данных.

Система управление базами данных (СУБД) = это программный комплекс, который позволяет управлять базами данных:
создавать их, извлекать из них данные, добавлять, изменять ил удалять эти данные.

Что такое транзакция?

Транзакция - это набор операций, которые предоставляют из себя одну логическую операцию. Поэтому транзакция может быть
выполнена либо целиком, либо не выполнена вообще.

Типичный пример - это банковская операция по переводу денег с одного счета на другой. По факту выполняется множество
различных операций: списать деньги со счета одного человека, зачислить эти деньги на счет другого человека,
отправить обоим оповещение о совершение сделки.

Набор этих операций - это одна логическая операция по переводу денег, то есть транзакция. И она может быть выполнена
только целиком. Если происходит какая-то ошибка, то вся транзакция отменяется. Процесс отмены транзакции
называется rollback (или откат).

Понятие ACID.

ACID - это набор требований к СУБД, которые обеспечивают правильное хранение и обработку данных.

Atomicity (или атомарность) - гарантирует, что каждая транзакция будет выполнена полностью или не будет выполнена
совсем. Не допускаются промежуточные состояния.

Consistency (или согласованность) - гарантирует, что каждая успешная транзакция по определению фиксирует только
допустимые результаты.

Isolation (или изолированность) - гарантирует, что во время выполнения транзакции не должны оказывать влияние на ее
результаты.

Durability (или надежность) - гарантирует, что независимо от проблем на нижних уровнях (обесточивание системы или сбоя
в оборудовании), изменения сделанные успешно завершенной транзакцией, должны остаться сохраненными после восстановления
системы.

Изолированность - требование очень дорогое, поэтому в реальных СУБД существуют разные режимы, которые позволяют
нам контролировать степень изолированности.

Понятия: таблица, колонка, строка.

Table (или таблица) - это основной объект БД, предназначенный для хранения данных в структурированном виде.

Row (или строка, запись) - совокупность, данных одного логического элемента БД.

Column (или колонка) - хранит значение все строк определенного типа.


Понятия: первичный и внешний ключ.

Primary key (PK или первичный ключ) - это специальный столбец, который содержит уникальные значения для каждой записи
в таблице.

Foreign Key (FK или внешний ключ) - это специальный столбец, который содержит значения PK из другой таблицы и задает
связь между записями этих таблиц.

Реляционные vs. Нереляционные БД.

Реляционные базы данных (или SQL базы данных) хранят данные в табличном формате со строками и столбцами. Строки могут
иметь связи (или отношения, relation-ы) с другими строками в других таблицах.
Самые популярные:
    *PostgreSQL.
    *MySQL.
    *Oracle DB.

Нереляционные базы данных (или NoSQL базы данных) используют НЕ систему строк и столбцов, а модель, которая
оптимизирована для хранения определенного типа содержимого. Например, данные могут храниться в виде документов
JSON, графов, а также записей формата "ключ-значение".
Самые популярные:
    *MongoDB.
    *Redis.
    *Cassandra.
