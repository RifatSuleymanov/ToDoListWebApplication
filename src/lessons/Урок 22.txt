Модуль: Spring Boot;
Лекция: Знакомство со Spring Boot Initializer;


Что такое Spring Boot Initializer?
Spring Boot Initializer - это специальный сервис, который позволяет нам с легкостью сгенерировать шаблонный и уже
сконфигурированный Spring Boot проект, готовый к использованию.

Сгенерировать такой проект мы можем либо через специальный сайт https://start.spring.io, либо напрямую через
IntelliJ IDEA.
Зависимости:
    1. Spring Web
    2. Spring Data JPA
    3. PostgreSQLDriver
    4. ThymeLeaf

Исследуем сгенерированный проект.
    1. Файл pom.xml имеет всего несколько зависимостей, большая часть из которых - это стартеры.
        *Стартер - это набор взаимосвязанных зависимостей.
        *Стартер Parent руководит всеми остальными стартерами. К примеру, именно у стартера Parent указывается версия
        которая будет использоваться во всех остальных стартерах.
    2. В файле pom.xml отсутствует упаковка веб-приложения в war.
    3. Присутствует встроенный в проект maven-wrapper.
    4. Присутствует сконфигурированная папка resources.
    5. Присутствует единственный класс с main-методами.

Аннотация @SpringBootApplication;
Аннотация @SpringBootApplication - это специальная аннотация, которая заменяет несколько других более примитивных
аннотаций, часть из которых мы уже использовали при конфигурации веб-приложения.
Составляющая аннотации @SpringBootApplication:
    1. @Configuration.
    2. @EnableAutoConfiguration.
    3. @ComponentScan.

Почему мы раньше не использовали Spring Boot?
    1. Чтобы на контрасте оценить, насколько сильно Spring Boot облегчает разработку веб-приложений.
    2. Для того чтобы было полное представление о том, как "под капотом" работает Spring Boot и его различные процессы.
