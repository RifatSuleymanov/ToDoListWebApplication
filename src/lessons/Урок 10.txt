Что такое DTO?
POJO (Plain Old Java Object) - это простой Java-объект, не унаследованный от какого-то специфического объекта
и не реализующий никаких служебных интерфейсов, кроме тех, которые нужны для бизнес-модели.

Entity - это POJO-объект, который хранит данные, используемые на уровне доступа к данным.

DTO (Data Transfer Object) - это POJO-объект, который хранит данные, используемые на уровне представления.

Model vs. ModelAndView.

Model.
    1. Всегда указывается в качестве аргумента метода.
    2. Можно настраивать только данные для отображения.


ModelAndView.
    1. Всегда указывается в качестве возвращаемого значения метода.
    2. Можно настраивать данные для отображения и само отображение.
    3. Объект нужно создавать вручную.

Альтернатива @RequestParam.
Если контроллер принимает несколько GET-параметров, то их можно в явном виде указать при помощи аннотации @RequestParam.

Проблема: если таких параметров становится очень много, то огромное количество, помеченных аннотацией @RequestParam,
будет вносить беспорядок.

Решение: можно создать какой-нибудь DTO и указать его в качестве аргумента метода контроллера. Тогда Spring сам
сформирует этот объект из всех GET-параметров.

    public class QueryParameters {
        private int id;
        private String filter;

        public QueryParameters(int id, String filter) {
            this.id = id;
            this.filter = filter;
        }

        public int getId() {
            return id;
        }

        public void setId(int id) {
            this.id = id;
        }

        public String getFilter() {
            return filter;
        }

        public void setFilter(String filter) {
            this.filter = filter;
        }
    }
